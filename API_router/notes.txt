API Router:
APIRouter is like a mini FastAPI application.
FastAPI App
│
├── Student Router
├── Course Router
├── User Router
├── Auth Router
└── Product Router
Each router owns a related set of endpoints. Finally, the main application combines them.

Best Practices
- Keep main.py focused on application setup.
- Group routes by feature, not by HTTP method.
- Use prefixes to create meaningful URLs.
- Use tags to organize Swagger documentation.
- Name routers with plural nouns when representing collections (users.py, products.py).
